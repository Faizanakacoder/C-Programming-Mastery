# Theory - Comments in C

## Introduction

Comments are explanatory notes written inside a program.

They are ignored by the compiler and do not affect the execution of a program.

Comments make programs easier to understand.

---

## Why Use Comments?

Comments are used to:

- Explain code
- Improve readability
- Help debugging
- Make maintenance easier
- Help other programmers understand code

---

## Types of Comments in C

### 1. Single Line Comment

A single line comment begins with `//`.

### Syntax

```c
// This is a single line comment
```

### Example

```c
// Displaying message
printf("Hello World");
```

---

### 2. Multi Line Comment

A multi line comment begins with `/*` and ends with `*/`.

### Syntax

```c
/*
This is
a multi line
comment
*/
```

### Example

```c
/*
This program
prints a message
*/
printf("Hello");
```

---

## Advantages of Comments

- Makes code understandable
- Improves readability
- Helps in teamwork
- Useful for documentation

---

## Real Life Example

Think of comments like notes in a notebook.

They help explain things but are not part of the final answer.

---

## Conclusion

Comments are important for writing clean, understandable, and professional programs.
